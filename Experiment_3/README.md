# Experiment 3: Jenkins + GCP Vertex AI Training Automation

**Name:** Soham | **Subject:** MLOps Lab | **Date:** April 2026

## Aim
To study model training automation using Jenkins CI/CD and GCP Vertex AI.

## Objective
- Understand Jenkins architecture and Declarative Pipeline syntax
- Learn GCP Vertex AI Custom Training Jobs
- Trigger cloud ML training from a local Jenkins server
- Monitor training and register models in Vertex AI Model Registry

## Theory

Jenkins is an open-source automation server running locally. A Jenkinsfile defines stages in Groovy.

GCP Vertex AI provides scalable cloud compute for custom ML training. Your Docker container runs on cloud VMs, saves the model to Google Cloud Storage (GCS), and registers to Vertex AI Model Registry.

Integration Flow:
Code Push to GitHub
  -> Jenkins detects (SCM polling every 5 min)
    -> Lint and Unit Tests
      -> Docker Build + Push to Artifact Registry
        -> Submit Vertex AI Custom Training Job
          -> Monitor until complete
            -> Register model in Vertex AI Model Registry
