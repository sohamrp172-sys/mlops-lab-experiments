# Experiment 5: Containerizing ML API using Docker with Kubernetes

**Name:** Soham | **Subject:** MLOps Lab | **Date:** April 2026

## Aim
To study containerization of an ML inference API using Docker and orchestration using Kubernetes (AKS).

## Objective
- Write a production-grade multi-stage Dockerfile for an ML API
- Push Docker images to Azure Container Registry (ACR)
- Deploy to Azure Kubernetes Service (AKS) with 3 replicas
- Configure liveness/readiness probes and Horizontal Pod Autoscaler (HPA)

## Theory

Why containers for ML: Docker packages code + runtime + dependencies into a portable unit. Ensures identical behavior on dev, staging, and production.

Kubernetes concepts:
- Pod: smallest deployable unit, contains container(s)
- Deployment: manages desired number of pod replicas
- Service: exposes pods to network (LoadBalancer type gives public IP)
- HPA: auto-scales pods from 3 to 10 based on CPU utilization

## Dockerfile (multi-stage)

    FROM python:3.10-slim as builder
        WORKDIR /app
            RUN apt-get update && apt-get install -y gcc && rm -rf /var/lib/apt/lists/*
                COPY requirements.txt .
                    RUN pip install --no-cache-dir --user -r requirements.txt

                        FROM python:3.10-slim
